    Transaction Control Language (TCL) in SQL¶

    üîπ What is TCL?¶

    TCL commands manage transactions in the database.
    They ensure data integrity and control how changes are saved or undone.

    üîπ Key TCL Commands (in depth)¶

    1. COMMIT¶

    • Saves changes permanently.
    COMMIT;

    2. ROLLBACK¶

    • Undo changes since last commit.
    ROLLBACK;

    3. SAVEPOINT¶

    • Creates a point to rollback partially.
    SAVEPOINT sp1;
ROLLBACK TO sp1;

    4. SET TRANSACTION¶

    • Sets transaction properties like isolation level.
    SET TRANSACTION READ ONLY;

    üîπ Pros & Cons of TCL¶

    ‚úÖ Pros¶

    1. Transaction Safety ‚Üí Prevents accidental data loss.
    2. Consistency ‚Üí Maintains ACID properties.
    3. Flexibility ‚Üí Can rollback fully or partially.
    4. Control ‚Üí Developers control when to save changes.

    ‚ùå Cons¶

    1. Complexity ‚Üí Misuse can cause locks.
    2. Performance ‚Üí Frequent rollbacks may reduce speed.
    3. Overhead ‚Üí Managing multiple savepoints may be costly.
    4. Limited Scope ‚Üí Works only with DML changes.
