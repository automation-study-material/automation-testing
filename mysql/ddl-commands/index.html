    Data Definition Language (DDL) in SQL¶

    üîπ What is DDL?¶

    DDL commands are used to define, modify, and manage the structure of database objects (like tables, views, schemas, indexes).

    These commands work at the schema level (structure), not on individual rows of data.

    üîπ Key DDL Commands (in depth)¶

    1. CREATE¶

    • Used to create database objects such as tables, schemas, views, or indexes.
    • You must define the structure (columns, data types, constraints).

    Example 

    CREATE TABLE Employee (
    EmpID INT PRIMARY KEY,
    Name VARCHAR(100) NOT NULL,
    Salary DECIMAL(10,2) CHECK (Salary > 0),
    DepartmentID INT,
    HireDate DATE DEFAULT CURRENT_DATE
);

    2. ALTER¶

    • Used to modify existing objects (add/remove/rename columns, change data types, add constraints).

    Example 

    -- Add a new column
ALTER TABLE Employee ADD Age INT;

-- Modify column type
ALTER TABLE Employee MODIFY Salary DECIMAL(12,2);

-- Drop a column
ALTER TABLE Employee DROP COLUMN Age;

-- Add a foreign key
ALTER TABLE Employee
ADD CONSTRAINT fk_dept FOREIGN KEY (DepartmentID) REFERENCES Department(DeptID);

    3. DROP¶

    • Deletes a database object completely (table, view, database, index).
    • ‚ö†Ô∏è Removes structure and all data.

    Example 

    DROP TABLE Employee;
DROP DATABASE HR;

    4. TRUNCATE¶

    • Removes all rows from a table, but keeps the structure.
    • Much faster than DELETE (no logging of each row).
    • Cannot be rolled back in some DBs.

    Example 

    TRUNCATE TABLE Employee;

    5. RENAME¶

    • Renames a database object (varies slightly across RDBMS).

    Example (MySQL / Oracle) 

    RENAME TABLE Employee TO Staff;

    6. COMMENT¶

    • Add descriptions to database objects. (Supported in Oracle, PostgreSQL, etc.)

    Example 

    COMMENT ON TABLE Employee IS 'Stores employee details';
COMMENT ON COLUMN Employee.Salary IS 'Monthly Salary in INR';

    üîπ Pros & Cons of DDL¶

    ‚úÖ Pros¶

    1. Schema Control ‚Üí DDL provides strong control over database design (tables, keys, indexes).
    2. Integrity Enforcement ‚Üí Constraints (PRIMARY KEY, FOREIGN KEY, CHECK) ensure data accuracy.
    3. Performance ‚Üí With indexes and partitions defined via DDL, queries become faster.
    4. Automation Friendly ‚Üí You can version-control DDL scripts for CI/CD pipelines.
    5. Flexibility ‚Üí Easily modify structures as business needs evolve.

    ‚ùå Cons¶

    1. Destructive Nature
    2. DROP and TRUNCATE permanently remove data (can‚Äôt easily rollback).

    3. Small mistake ‚Üí huge data loss.

    4. Locking Issues

    5. Some DDL operations lock tables (no read/write allowed until completed).

    6. Dependent Objects

    7. Dropping/altering tables can break views, stored procedures, or queries that depend on them.

    8. Environment Differences

    9. Syntax varies across RDBMS (MySQL, Oracle, PostgreSQL). Example: ALTER column syntax differs.

    10. Not Transaction-Safe (in some DBs)

    11. In MySQL, some DDL operations auto-commit ‚Üí cannot be rolled back.

    üîπ Advantages Explained with Examples¶

    1. Schema Control¶

    DDL lets you define strong structures with relationships.

    CREATE TABLE Department (
    DeptID INT PRIMARY KEY,
    DeptName VARCHAR(50) UNIQUE
);

CREATE TABLE Employee (
    EmpID INT PRIMARY KEY,
    Name VARCHAR(100) NOT NULL,
    Salary DECIMAL(10,2),
    DeptID INT,
    FOREIGN KEY (DeptID) REFERENCES Department(DeptID)
);

    üëâ Employees must belong to a valid department.

    2. Integrity Enforcement¶

    Constraints enforce data accuracy.

    CREATE TABLE Employee (
    EmpID INT PRIMARY KEY,
    Name VARCHAR(100) NOT NULL,
    Salary DECIMAL(10,2) CHECK (Salary > 0),
    DeptID INT NOT NULL,
    FOREIGN KEY (DeptID) REFERENCES Department(DeptID)
);

    üëâ Prevents negative salaries and missing departments.

    3. Performance (Indexes & Partitions)¶

    Indexes and partitions improve query speed.

    -- Index for faster salary searches
CREATE INDEX idx_salary ON Employee(Salary);

-- Partition sales table by year
CREATE TABLE Sales (
    SaleID INT PRIMARY KEY,
    SaleDate DATE,
    Amount DECIMAL(10,2)
)
PARTITION BY RANGE (YEAR(SaleDate)) (
    PARTITION p2023 VALUES LESS THAN (2024),
    PARTITION p2024 VALUES LESS THAN (2025)
);

    üëâ Queries on salaries or specific years run faster.

    4. Automation Friendly (CI/CD)¶

    DDL scripts can be stored and executed in deployments.

    -- migration_v2.sql
ALTER TABLE Employee ADD COLUMN Email VARCHAR(100);

    üëâ Keeps dev, test, and prod in sync automatically.

    5. Flexibility (Evolving Structures)¶

    Easily adapt schema to new business needs.

    ALTER TABLE Employee ADD COLUMN PhoneNumber VARCHAR(15);
ALTER TABLE Employee RENAME COLUMN Name TO FullName;

    üëâ Business changes are easily handled.

    üìä DDL Commands ‚Äì Feature Comparison¶

    Command Rollback Possible Data Loss Structure Change Typical Use Case
    CREATE ‚úÖ Yes (before commit) ‚ùå No ‚úÖ Creates new object Create tables, views, indexes
    ALTER ‚úÖ Yes (before commit) ‚ùå No ‚úÖ Modifies structure Add/modify columns, constraints
    DROP ‚úÖ Yes (before commit) ‚úÖ Yes (all data lost) ‚úÖ Removes object completely Delete table/schema permanently
    TRUNCATE ‚úÖ Yes (before commit) ‚úÖ Yes (all rows deleted) ‚ùå No (structure remains) Quickly remove all records
    RENAME ‚úÖ Yes (before commit) ‚ùå No ‚úÖ Only name changes Rename table/column
    COMMENT ‚úÖ Yes (before commit) ‚ùå No ‚ùå No (metadata only) Add description to objects
    CREATE INDEX ‚úÖ Yes (before commit) ‚ùå No ‚úÖ Creates performance structure Improve query performance
    DROP INDEX ‚úÖ Yes (before commit) ‚ùå No (only index removed) ‚úÖ Removes performance structure Free up space, remove unused index

    üîë Key Notes¶

    • Rollback ‚Üí Possible only until you COMMIT (some DBs auto-commit DDL, e.g., Oracle).
    • Data Loss ‚Üí DROP & TRUNCATE cause data removal; ALTER/RENAME don‚Äôt.
    • Structure Change ‚Üí All DDL affects schema except COMMENT.
    • Performance Impact ‚Üí TRUNCATE is faster than DELETE; indexes speed queries but slow inserts/updates.
