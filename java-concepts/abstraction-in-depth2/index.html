    Abstraction in Java (Abstract Class & Interface ‚Äì Complete Interview Guide)¶

    üî∑ What is Abstraction?¶

    Abstraction is the process of hiding implementation details and exposing only essential behavior.

    It focuses on what an object does, not how it does it.

    üî∑ How Java Supports Abstraction¶

    Java provides abstraction using: 1. Abstract Classes 2. Interfaces

    üî∑ Abstract Class ‚Äì Core Concept¶

    abstract class Vehicle {
    abstract void start();   // abstract method

    void stop() {            // concrete method
        System.out.println("Vehicle stopped");
    }
}

    Key Points¶

    • Cannot be instantiated
    • Can contain abstract + concrete methods
    • Can have variables and constructors

    üî∑ Abstract Class Constructor (Important)¶

    Abstract classes DO have constructors.

    abstract class Vehicle {
    Vehicle() {
        System.out.println("Vehicle constructor");
    }
}

class Car extends Vehicle {
    Car() {
        System.out.println("Car constructor");
    }
}

    Vehicle v = new Car();

    Output¶

    Vehicle constructor
Car constructor

    üëâ Constructor is used to initialize common state.

    üî∑ JVM / Backend Working (Must-Know)¶

    Vehicle v = new Car();
v.start();

    Compile Time¶

    • Reference type (Vehicle) is checked
    • Method existence verified

    Runtime¶

    • JVM identifies actual object (Car)
    • Calls overridden method using Dynamic Method Dispatch

    ‚û°Ô∏è This is Runtime Polymorphism

    üî∑ Abstraction + Method Overriding¶

    Concept¶

    Overriding enables abstraction by forcing subclasses to provide specific behavior.

    abstract class Shape {
    abstract void draw();
}

    class Circle extends Shape {
    @Override
    void draw() {
        System.out.println("Drawing Circle");
    }
}

    Shape s = new Circle();
s.draw(); // Drawing Circle

    Interview Line¶

    Abstraction is implemented using method overriding at runtime.

    üî∑ Abstraction + Method Overloading (Important Difference)¶

    Overloading happens at compile time¶

    • Same method name
    • Different parameters
    abstract class Calculator {
    int add(int a, int b) {
        return a + b;
    }

    int add(int a, int b, int c) {
        return a + b + c;
    }
}

    üëâ Overloading does NOT achieve abstraction - It is compile-time polymorphism - Method selection based on reference type

    üî∑ Abstract Class vs Interface (Deep Comparison)¶

    Feature Abstract Class Interface
    Keyword abstract class interface
    Multiple inheritance ‚ùå No ‚úîÔ∏è Yes
    Constructors ‚úîÔ∏è Yes ‚ùå No
    Variables Any type public static final
    Methods Abstract + concrete Abstract, default, static
    State ‚úîÔ∏è Can maintain ‚ùå Cannot
    Use case Base class with code Pure contract

    üî∑ How Interface Achieves Abstraction¶

    interface Payment {
    void pay();
}

    class UpiPayment implements Payment {
    public void pay() {
        System.out.println("Paying via UPI");
    }
}

    Payment p = new UpiPayment();
p.pay();

    üëâ Interface provides 100% abstraction (before Java 8)

    üî∑ Abstract Class Implementing Interface¶

    interface Flyable {
    void fly();
}

    abstract class Bird implements Flyable {
    // may skip implementation
}

    class Sparrow extends Bird {
    public void fly() {
        System.out.println("Sparrow flying");
    }
}

    üî∑ Real-Time Interview Questions (With Theory + Code)¶

    Q1. Why can‚Äôt abstract class be instantiated?¶

    Theory:
    Because it may contain unimplemented methods.

    abstract class A {
    abstract void m1();
}

    A a = new A(); // Compile-time error

    Q2. Can abstract class have zero abstract methods?¶

    ‚úîÔ∏è Yes ‚Äì used to prevent instantiation.

    abstract class Utility {
    void log() {
        System.out.println("Logging");
    }
}

    Q3. Can constructor be abstract?¶

    ‚ùå No ‚Äì constructors cannot be overridden.

    Q4. What happens if child does not implement abstract method?¶

    ‚ùå Compile-time error (unless child is abstract).

    abstract class A {
    abstract void m1();
}

class B extends A { } // Error

    Q5. Can abstract class have static methods?¶

    ‚úîÔ∏è Yes
    ‚ùå Static methods cannot be abstract.

    Q6. Difference between abstraction and encapsulation?¶

    Abstraction Encapsulation
    Hides implementation Hides data
    Achieved using abstract/interface Achieved using private fields
    Design level Implementation level

    Q7. Which is preferred in real projects ‚Äì interface or abstract class?¶

    Answer: - Use interface for APIs and contracts - Use abstract class for base implementations

    (Spring heavily uses interfaces)

    Q8. Is abstraction compile-time or runtime?¶

    ‚úîÔ∏è Runtime
    (Abstract methods are resolved at runtime using overriding)

    üî∑ Common Interview Traps¶

    Trap 1¶

    abstract class A {
    abstract void m1();
}

class B extends A {
}

    ‚ùå Compile error
    ‚úîÔ∏è Fix: Implement method or make class abstract

    Trap 2¶

    abstract class Test {
    abstract static void run(); // ‚ùå Invalid
}

    üî∑ One-Line Interview Definitions¶

    • Abstraction: Hiding implementation details and exposing behavior
    • Abstract Class: Partial abstraction with base implementation
    • Interface: Full abstraction and contract definition
