    Playwright inputValue() & textContent() ‚Äî What They REALLY Return¶

    ‚úÖ Short Answer¶

    • inputValue() ‚Üí returns the current value property of form elements
    • textContent() ‚Üí returns the text inside an element

    They are not interchangeable and are used for different types of elements.

    üß† Correct Mental Model¶

    ‚ùå Incorrect understanding¶

    inputValue() returns what the user typed
    textContent() works for everything

    ‚úÖ Correct understanding¶

    inputValue() ‚Üí reads element.value (inputs, textareas, selects)
    textContent() ‚Üí reads inner text of normal elements

    üîπ inputValue() ‚Äî Explained¶

    ‚úÖ What it returns¶

    inputValue() returns the current value of the input field ‚Äî
    not how or who entered it.

    That value can come from: - User typing - Playwright (fill(), type()) - HTML default value - JavaScript updates

    Internally, Playwright reads:

    element.value

    üß™ inputValue() Examples¶

    1Ô∏è‚É£ User types manually¶

    <input id="name">

    User types Nitin

    await page.locator('#name').inputValue();
// "Nitin"

    2Ô∏è‚É£ Playwright fills the value¶

    await page.locator('#name').fill('Patil');

await page.locator('#name').inputValue();
// "Patil"

    3Ô∏è‚É£ Value set via HTML¶

    <input id="city" value="Pune">

    await page.locator('#city').inputValue();
// "Pune"

    4Ô∏è‚É£ Value changed via JavaScript¶

    await page.evaluate(() => {
  document.querySelector('#name').value = 'Mumbai';
});

    await page.locator('#name').inputValue();
// "Mumbai"

    ‚ùå What inputValue() Does NOT Do¶

    • Track keyboard events
    • Know who typed (user vs automation)
    • Read visible text
    • Read placeholders
    • Read inner text

    üîπ textContent() ‚Äî Explained¶

    ‚úÖ What it returns¶

    textContent() returns the text inside an element, including hidden text.

    It works for: - <div> - <span> - <p> - <label> - <button>

    ‚ùå It does NOT work for <input> or <textarea>

    Internally, Playwright reads:

    element.textContent

    üß™ textContent() Examples¶

    1Ô∏è‚É£ Reading visible text¶

    <div id="msg">Welcome Nitin</div>

    await page.locator('#msg').textContent();
// "Welcome Nitin"

    2Ô∏è‚É£ Hidden text is included¶

    <div id="status">
  Success
  <span style="display:none">Hidden</span>
</div>

    await page.locator('#status').textContent();
// "SuccessHidden"

    3Ô∏è‚É£ ‚ùå Common mistake¶

    await page.locator('#name').textContent(); // ‚ùå WRONG

    Reason:

    <input> elements do not have inner text ‚Äî only a value.

    üÜö inputValue() vs textContent()¶

    Feature inputValue() textContent()
    Used for Input elements Normal elements
    Reads element.value element.textContent
    Works on <input> ‚úÖ Yes ‚ùå No
    Includes hidden text N/A ‚úÖ Yes
    Tracks typing ‚ùå No ‚ùå No

    üÜö inputValue() vs getAttribute('value')¶

    <input id="email" value="initial@mail.com">

    User types changed@mail.com

    Method Returned Value
    inputValue() changed@mail.com
    getAttribute('value') initial@mail.com

    ‚úÖ Best Practice in Playwright (Recommended)¶

    await expect(page.locator('#email')).toHaveValue('changed@mail.com');
await expect(page.locator('#msg')).toHaveText('Welcome Nitin');

    üß† Easy Rule to Remember¶

    User enters data ‚Üí inputValue()
    User sees text ‚Üí textContent()

    üìå Final Answer¶

    inputValue() returns the current value property of form fields.
    textContent() returns the text inside normal DOM elements.
