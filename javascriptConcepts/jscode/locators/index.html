    Playwright Locators -- Complete Guide (JavaScript)¶

    This guide covers all locator strategies in Playwright (JavaScript) and explains when and why to use each type.

    üîπ Example Base HTML¶

    <input type="text" id="email" class="form-control" placeholder="Enter email" />

<button id="submitBtn" class="btn btn-primary" type="submit">Submit</button>

<span class="text">Title</span>

<a href="/home">Home</a>

    üîπ Locator Types (With Context + HTML + JS Example)¶

    1Ô∏è‚É£ Locate by ID¶

    IDs are unique within a page, making them the most reliable and fastest selectors.

    üî∏ HTML:¶

    <input id="email" />
<button id="submitBtn">Submit</button>

    üî∏ Playwright:¶

    page.locator("#email");
page.locator("#submitBtn").click();

    2Ô∏è‚É£ Locate by Class¶

    Class selectors are useful when elements share styling or behavior.

    üî∏ HTML:¶

    <input class="form-control" />
<button class="btn btn-primary">Save</button>

    üî∏ Playwright:¶

    page.locator(".form-control");
page.locator(".btn.btn-primary").click();

    3Ô∏è‚É£ Locate by Tag Name¶

    Used when selecting elements by their tag type.

    üî∏ HTML:¶

    <button>Submit</button>
<input type="text" />

    üî∏ Playwright:¶

    page.locator("button");
page.locator("input");

    4Ô∏è‚É£ Locate by Text Content¶

    Useful for selecting elements based on visible text.

    üî∏ HTML:¶

    <button>Submit</button>
<a>Home</a>

    üî∏ Playwright:¶

    page.getByText("Submit").click();
page.getByText("Home");

    5Ô∏è‚É£ Locate by ARIA Role (Recommended)¶

    This is Playwright's most powerful and stable locator type.

    üî∏ HTML:¶

    <button type="submit">Submit</button>
<a href="/home">Home</a>

    üî∏ Playwright:¶

    page.getByRole("button", { name: "Submit" }).click();
page.getByRole("link", { name: "Home" });

    6Ô∏è‚É£ Locate by Placeholder¶

    Great for selecting input elements using placeholder text.

    üî∏ HTML:¶

    <input placeholder="Enter email" />

    üî∏ Playwright:¶

    page.getByPlaceholder("Enter email").fill("abc@test.com");

    7Ô∏è‚É£ Locate by Label¶

    Automatically links <label> to its corresponding <input>.

    üî∏ HTML: (Applicable)¶

    • It will work onlu when there is association with label field see below example.
    • here you can see email is linked between label and input id.
    • In some cases input field is wrapped in labes.
    <label for="email">Email</label>
<input id="email" />

    üî∏ HTML: (NOT Applicable)¶

    • It will not work when there is NO association with label and input field see below example.
    div class="form-group">
    <label>Name</label>
       <input class="form-control ng-untouched ng-pristine ng-invalid" minlength="2" name="name" required="" type="text">
</div>

    üî∏ Playwright:¶

    page.getByLabel("Email").fill("test@domain.com");

    8Ô∏è‚É£ Locate by Alt Text¶

    Useful for images.

    üî∏ HTML:¶

    <img src="logo.png" alt="Company Logo">

    üî∏ Playwright:¶

    page.getByAltText("Company Logo");

    9Ô∏è‚É£ Locate by Title Attribute¶

    Handy when elements do not have visible text.

    üî∏ HTML:¶

    <button title="Click to submit">Submit</button>

    üî∏ Playwright:¶

    page.getByTitle("Click to submit").click();

    üîü Locate using CSS Attribute Selectors¶

    üî∏ HTML:¶

    <input type="text" />
<button type="submit">Save</button>
<a href="/home">Home</a>

    üî∏ Playwright:¶

    page.locator("input[type='text']");
page.locator("button[type='submit']");
page.locator("a[href='/home']");

    1Ô∏è‚É£1Ô∏è‚É£ Locate using XPath¶

    üî∏ HTML:¶

    <input id="email" />
<button>Submit</button>

    üî∏ Playwright:¶

    page.locator("//input[@id='email']");
page.locator("//button[text()='Submit']");

    üîπ Handling Multiple Elements¶

    üî∏ HTML:¶

    <ul>
  <li class="item">Apple</li>
  <li class="item">Banana</li>
  <li class="item">Cherry</li>
</ul>

    üî∏ Playwright:¶

    const items = page.locator(".item");
const count = await items.count();

await items.nth(0).innerText();
await items.nth(1).innerText();
await items.nth(2).innerText();

for (let i = 0; i < count; i++) {
  console.log(await items.nth(i).innerText());
}

const banana = page.locator(".item").filter({ hasText: "Banana" });

page.locator(".item").first();
page.locator(".item").last();

    üîπ Summary¶

    • Prefer semantic locators ‚Üí getByRole, getByLabel, getByPlaceholder
    • Use CSS/XPath only when needed
    • Use .nth(), .first(), .last(), .filter() for multi-elements
