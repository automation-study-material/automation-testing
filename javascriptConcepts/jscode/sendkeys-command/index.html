    ‚úî Important Points - Clears existing text automatically - Inserts value instantly (not typed) - Does NOT trigger keydown/keyup for each character - Only final input/change events fire - Fastest common method

    ‚úî Important Points - ‚ùó Does NOT clear existing text ‚Üí it appends - Types characters one by one - Triggers keydown ‚Üí keypress ‚Üí keyup for every character - Works like a human typing - Supports typing delays

    ‚úî Important Points - Same as type(), but uses keyboard, not locator - Requires clicking or focusing element first - Triggers full key events - Does not clear text

    ‚úî Important Points - Sends one key press, not text - Useful for shortcuts: Enter, Backspace, Tab, ArrowUp - Triggers full keydown/keyup events - Does not clear text

    5Ô∏è‚É£ keyboard.insertText() ‚Äì Insert Text Without Keystrokes¶

    ‚úî Important Points - Does NOT trigger any key events - Does not simulate typing - Does not clear text - Inserts text directly into DOM - Fastest method when keystrokes not needed

    ‚úî Example

    ‚úî When to Use - Avoiding event listeners - Speed/performance testing - Inputs that block simulated typing

    6Ô∏è‚É£ evaluate() ‚Äì Set Value Using JavaScript¶

    ‚úî Important Points - Uses page JS context to set .value - Does not fire key events - Does not simulate typing - Completely bypasses frontend validations - Clears existing text only if you set empty string

    ‚úî Example

    ‚úî When to Use - When UI blocks typing/filling - React/Angular controlled inputs - Special custom editors

    üîç Detailed Comparison¶

    Clearing Behavior¶

    • ‚úî fill() clears text
    • ‚úî evaluate() can clear if set to ""
    • ‚ùå Others do not clear

    Typing Simulation¶

    Only these behave like real typing: - type() - keyboard.type()

    Trigger Events¶

    • type() / keyboard.type() ‚Üí all key events
    • fill() ‚Üí only final input/change
    • insertText() ‚Üí no key events
    • evaluate() ‚Üí no events

    Best for Reliability¶

    Scenario Best Method
    Entering text normally fill()
    Simulating human typing type()
    Sending keys (Enter, Tab) press()
    Bypassing event handlers insertText()
    UI not accepting input evaluate()
